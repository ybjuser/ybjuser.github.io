#

js高级十个知识点

标签（空格分隔）： 知识总结

1.函数定义方式不同，预解析的结果会不同。
由于var,(2)中的fun函数在预解析时，会作为一个变量名处理，所以其初始值是undefind;并不能作为一个函数名进行调用。

(1)fun();       //弹出框123                  
     function fun(){
         alert(“123”);
     }
(2)fun();       //报错
     var fun = function(){
  alert(“123”);
 }

2.undefind和is not defind
用var定义的变量a会进行预解析，初始值是undefind,而未定义的b直接使用，属于程序错误，会报错。

alert(a);   //结果为undefind
 var a=4;  
alert(b);   //结果为报错 is not defind.

3.预解析会按照

5.函数的定义会对预解析产生隔离区
和局部变量类似，定义在函数内部的变量，预解析的结果也只能在函数内部使用，不能在它的上一级以及全局中使用，所以在全局中的alert(num)会报错。

function show(){
    alert(num);  //结果为undefind
    var num = 4;
}
show();
alert(num);  //结果为报错

6.当变量的名字和函数的名字相同时，预解析的结果。
预解析时，变量的初始值是undefind，函数的初始值是函数体本身。当两个名字相同时，预解析也会出现覆盖现象。所以第一个console.log(fun)输出的是函数体本身。
当有赋值=、++和–等，会对初始值进行改变。所以在第二个console.log(fun)输出的是 var fun = 2; 改变了fun后的值。

console.log(fun);     //输出函数体本身
var fun = 2;
function fun(){
    alert("调用");
}
console.log(fun);      //输出2

7.预解析针对的
没有var声明，会认为是对一个变量的值进行改变，不会进行预解析。所以不会对x 进行预解析。console.log(x) 报错。

console.log(x);  //报错
x = 3;

8.作用域链
本级作用域只能往它上一级作用域里寻找变量，不能往它的下一集查找;当在本级或者上一级找到这个变量时，就停止查找；如果一级一级向上查找，仍然没有找到这个变量，说明变量未定义，程序报错：is not defind

var x = 3;
function f1(){
    var y = 4;
    function f2(){
        var z = 5;
        console.log(x + y + z);  //13
    }
    console.log(x + y);  //7
    f2();
    console.log(x + y + z);  //z is not defind
}
f1();
console.log(x + y);  //报错：y is not defind
f2();  //报错：is not defind

9.作用域链确定的时间
作用域是在定义的时候确定的，而不是调用运行的时候。所以在f函数中返回的匿名函数在的作用域在定义的时候就已经确定为：它自己 –> f 作用域 –> 全局作用域。所以最后返回的b的值是”bbb”。

function f(){
    var b = "bbb";
    return function(){  
        return b;
    }
}
var out = f();  // out 的值是f返回的匿名函数
var result = out();  //result 的值是b
console.log(result);

10.ES6
(1)在ES6中let实现了块级作用域，定义的变量x 只能在{}内使用。

{
let x = 1;
var y = 2;
}
console.log(x);   //报错
console.log(y);   //输出2

(2)let变量提升问题
let定义的变量不会提升，必须先声明后使用。

alert(a);  //报错：a is not defind
let a = 2;

(3)let认为每一轮循环都是独立的块
用let声明的i只在当前一轮有效，每一轮的i都是独立的

var a = [];
for(let i = 0 ; i < 3 ; i++){
    a[i] = function(){
        console.log(i);
    }
}
a[0]();    //输出0
a[1]();    //输出1

(4)let变量重复声明
var可以重复声明，发生覆盖；let重复声明会报错，必须严格要求，只能声明一次。

{
    let w = "aa";
    let w = "bb";
    alert(w);   //报错 
}

(5)const
const定义的相当于常量，其值不能改变，只读。所以在对a进行修改时会报错。

const a = 2;
console.log(a);
a = 3;      //报错
console.log(a);

11.使用构造函数+原型方式创建的对象，访问其属性或方法的顺序是：（1）先在自己的构造函数寻找；（2）找不到去原型空间

function Dog(name,color){
    this.name = name;
}
Dog.prototype.name = "露西";
Dog.prototype.color = "black";
var dog = new Dog("泰迪","gray");
console.log(dog.name);  //泰迪
console.log(dog.color);   //black

12.继承和多态
个性 > 共性
call()参数1：新绑定的this对象
参数1之后，依次为原来的函数的形参对应的实参
call是用来改变某个函数的this指向的

function Animal(n,s){
           this.type = n;
           this.sex = s;
   }
   Animal.prototype.speak = function(){    //共性
       alert(this.type);
   }
   function Bird(t,s,w){        
       Animal.call(this,t,s);       
   this.fly = w;            
   } 
   Bird.prototype.speak = function(){   //个性
       alert("这是bird自己定义的speak方法");
   }
   bird.speak();  // 弹出后面自己定义的方法
